# 外围系统设计文档

## 概述

外围系统包含存档、成就、新手引导和设置等非核心 RPG 玩法但对游戏完整度重要的系统。

---

# 第一部分：存档系统

## 设计

- 3 个存档槽，完全隔离
- 存档结构：`data/save_{slot}/player.json` + `{npc_id}.json`
- 游戏每 5 秒自动保存位置
- 切换存档时自动清空并重新加载所有数据

## 存档内容

- 玩家属性（等级、经验、HP/MP、属性、装备、背包、金币）
- 已学技能和天赋
- 任务进度（active/completed）
- NPC 好感度
- 位置信息（当前地图、坐标）

# 第二部分：成就系统（规划）

## 设计目标

长期目标追踪，增加游戏粘性和收集乐趣。

## 成就类别

| 类别 | 示例成就 | 条件 |
|------|----------|------|
| 战斗 | 初战告捷 | 首次击杀怪物 |
| 战斗 | 百人斩 | 累计击杀 100 只怪物 |
| 战斗 | 屠龙者 | 击杀骷髅王 |
| 探索 | 探险家 | 发现所有地图 |
| 探索 | 宝藏猎人 | 打开 20 个宝箱 |
| 社交 | 万人迷 | 任意 NPC 好感度达到 100 |
| 社交 | 交友广泛 | 所有 NPC 好感度超过 50 |
| 收集 | 装备大师 | 收集 50 种不同装备 |
| 收集 | 药剂师 | 使用 100 瓶药水 |
| 成长 | 满级 | 达到等级 20 |
| 经济 | 富甲一方 | 持有 10000 金币 |

## 成就数据结构

```json
{
  "achievement_id": "first_kill",
  "name": "初战告捷",
  "description": "首次击杀怪物",
  "category": "combat",
  "condition": {"type": "kill_count", "value": 1},
  "reward": {"title": "新手冒险者", "exp": 50}
}
```

## 成就奖励

| 奖励类型 | 说明 |
|----------|------|
| 称号 | 显示在角色名旁的称号 |
| 经验值 | 一次性经验奖励 |
| 物品 | 特殊物品（装饰性或实用性） |

# 第三部分：新手引导系统（规划）

## 设计目标

帮助新玩家理解游戏机制，降低上手门槛。

## 引导流程

| 步骤 | 内容 | 触发条件 |
|------|------|----------|
| 1 | 移动教学（WASD） | 首次进入游戏 |
| 2 | 交互教学（E 键与 NPC 对话） | 到达村庄中心 |
| 3 | 任务教学（接取第一个任务） | 与铁匠对话 |
| 4 | 背包教学（查看获得的物品） | 完成第一个任务目标 |
| 5 | 战斗教学（攻击、防御、技能） | 首次遭遇怪物 |
| 6 | 装备教学（装备武器提升属性） | 获得第一件装备 |
| 7 | 商店教学（买卖物品） | 回到村庄 |

## 引导 UI

- 高亮提示：半透明遮罩 + 高亮目标区域
- 文字气泡：NPC 头顶显示引导文字
- 进度指示：引导步骤列表
- 跳过按钮：老玩家可跳过引导

# 第四部分：设置系统

## 基本设置

| 设置项 | 选项 |
|--------|------|
| 音乐音量 | 0-100 滑块 |
| 音效音量 | 0-100 滑块 |
| 自动保存间隔 | 5秒 / 10秒 / 30秒 / 关闭 |
| 文字速度 | 慢 / 中 / 快 |
| 语言 | 中文 / English（预留） |

---

# 第五部分：死亡与复活机制（规划）

- 战斗失败在村庄神殿复活
- 死亡惩罚：5% 金币 + 10% 经验
- HP 恢复为 1，MP 恢复为满

# 操作指南

| 按键 | 功能 |
|------|------|

| E | 与 NPC/物件/怪物交互 |

| P | 打开角色信息 |
| Q | 打开任务管理 |
| T | 打开天赋面板 |
| O | 游戏菜单 |
| H | 帮助面板 |

|------|------|
