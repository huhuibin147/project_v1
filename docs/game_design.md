# 游戏设计文档

## 概述

LLM NPC 像素风 RPG 是一个基于 Web 的像素风角色扮演游戏。玩家在村庄和森林中探索，与 LLM 驱动的 NPC 自由对话、交易物品、战斗怪物。

| 层级 | 技术 |
|------|------|
| 前端 | HTML5 Canvas + 原生 JavaScript |
| 后端 | Python FastAPI |
| LLM | OpenAI 兼容协议（支持 DeepSeek、通义千问、Ollama 等） |

## 架构设计

```
┌─────────────────────────────────────────────────┐
│                   前端 (Canvas + DOM)             │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
│  │ 地图  │ │ 玩家  │ │ NPC  │ │ 战斗  │ │ UI   │  │
│  └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘  │
│     └────────┴────────┴────────┴────────┘       │
│                    fetch API                     │
├─────────────────────────────────────────────────┤
│                   后端 (FastAPI)                  │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
│  │ 地图  │ │ 玩家  │ │ NPC  │ │ 战斗  │ │ 物品  │  │
│  └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘  │
│     └────────┴────────┴────────┴────────┘       │
│              JSON 文件持久化 (data/)              │
├─────────────────────────────────────────────────┤
│              LLM API (OpenAI 兼容)               │
└─────────────────────────────────────────────────┘
```

### 前端架构

- **渲染**：Canvas 逐像素绘制，无外部图片资源
- **游戏循环**：`requestAnimationFrame`，每帧 `update(dt)` + `render()`
- **摄像机**：跟随玩家居中，边界钳制，视口裁剪优化
- **深度排序**：NPC、怪物、玩家按 Y 坐标排序绘制，实现遮挡
- **UI 面板**：DOM 元素覆盖在 Canvas 上，`.active` 类控制显示
- **互斥机制**：所有面板共享 `combatOpen`/`dialogueOpen`/`inventoryOpen` 等标志

### 后端架构

- **单例玩家**：全局 `PlayerProfile` 实例，所有操作自动存档
- **懒加载 NPC**：首次访问时创建，存档切换时重建
- **内存战斗**：`CombatSession` 仅存内存，10 分钟过期自动清理
- **配置驱动**：物品、NPC、怪物、技能、地图均为 JSON 配置
- **技能系统**：`skill_system.py` 管理技能定义、学习条件校验、格式化输出

## 玩家系统

### 职业

| 职业 | HP | MP | 攻击 | 防御 | 速度 | 定位 |
|------|-----|-----|------|------|------|------|
| 战士 | 120 | 30 | 15 | 12 | 8 | 高血量高防御 |
| 盗贼 | 80 | 50 | 18 | 6 | 15 | 高攻高速 |
| 法师 | 70 | 100 | 22 | 5 | 10 | 高攻低防高MP |

### 属性计算

```
基础属性 = 职业基础 + (等级-1) × 成长值
总属性   = 基础属性 + 装备加成

成长值：HP +10, MP +3, 攻击 +3, 防御 +2, 速度 +1（每级）
```

### 魔法值 (MP) 系统

- **MP**：魔法值，用于释放技能
- **基础 MP**：战士 30，盗贼 50，法师 100
- **MP 成长**：每级 +3
- **MP 恢复**：
  - 非战斗状态：每 10 秒恢复 5% 最大 MP
  - 升级时：MP 回满
  - 使用魔力药水：立即恢复 MP
- **MP 显示**：HUD 状态条、角色信息面板、战斗面板均显示当前 MP / 最大 MP

### 等级系统

- 经验值累积触发升级
- 升级所需经验递增（×1.5）
- 升级后自动重新计算属性（HP/MP 回满）
- 经验来源：战斗胜利、任务完成（预留）

### 存档系统

- 3 个存档槽，完全隔离
- 存档结构：`data/save_{slot}/player.json` + `{npc_id}.json`
- 游戏每 5 秒自动保存位置
- 切换存档时自动清空并重新加载所有数据

## 技能系统

### 概述

技能是消耗 MP 释放的特殊能力，可在战斗中使用。每个职业有专属技能，通过技能书学习新技能。

### 技能数据结构

```json
{
  "skill_id": "fireball",
  "name": "火球术",
  "description": "发射火球造成魔法伤害",
  "mp_cost": 15,
  "cooldown": 0,
  "type": "damage",
  "target": "enemy",
  "damage_type": "magic",
  "power": 1.5,
  "effects": [{"type": "burn", "duration": 3, "chance": 0.3}],
  "class_requirement": ["mage"],
  "level_requirement": 1
}
```

| 字段 | 说明 |
|------|------|
| `skill_id` | 唯一标识 |
| `name` | 显示名称 |
| `description` | 技能描述 |
| `mp_cost` | MP 消耗 |
| `cooldown` | 冷却回合数 |
| `type` | 技能类型：damage / heal / buff / debuff |
| `target` | 目标：enemy / self / ally |
| `damage_type` | 伤害类型：magic / physical |
| `power` | 威力倍率（× 攻击力） |
| `effects` | 附加效果列表 |
| `class_requirement` | 职业限制（空数组表示通用） |
| `level_requirement` | 等级要求 |

### LLM 驱动对话

NPC 由大语言模型驱动，每次对话构建包含以下上下文的 Prompt：

```
System: 角色设定 + 性格参数 + 当前情绪/好感度 + 商店库存 + 玩家背包 + 对话历史（最近10轮）
User: 玩家输入
→ LLM 返回结构化 JSON
```

**LLM 输出格式**：
```json
{
  "reply": "NPC 对话（1-3句）",
  "intent": "chat|quest|trade|unknown",
  "mood": "当前情绪",
  "affinity_change": -10到+10,
  "trade_action": null | {"action": "buy|sell", "item_id": "...", "quantity": N}
}
```

### 性格系统

每个 NPC 有 4 个性格参数（0-1），影响 LLM 输出风格：

| 参数 | 说明 |
|------|------|
| friendliness | 友善度 |
| courage | 勇气 |
| greed | 贪婪度 |
| humor | 幽默感 |

### 好感度与情绪

- 好感度范围 0-100，初始值由 NPC 配置定义
- 玩家言行（夸赞、侮辱、帮助）影响好感度变化
- 情绪随对话动态变化（平静/高兴/担忧/生气）
- 好感度影响 NPC 态度和交易行为

### 意图识别

LLM 自动分类玩家意图：

| 意图 | 触发条件 | 行为 |
|------|----------|------|
| chat | 日常对话 | NPC 自由回复 |
| quest | 询问任务 | NPC 描述任务（预留） |
| trade | 明确说"买"/"卖" | 自动执行交易 |
| unknown | 无法识别 | NPC 礼貌回应 |

### 交易系统

**两种交易路径**：
1. **对话驱动**：玩家在对话中表达交易意图，LLM 识别后自动执行
2. **商店面板**：按 E 打开 NPC 交互选项，选择"商店"直接交易

**交易流程**：
- 购买：校验玩家金币 → 扣除金币 → 物品加入背包
- 出售：校验玩家物品 → 扣除物品 → 金币加入背包
- NPC 收购的物品存入独立的收购仓库，不混入商店库存

## 战斗系统

### 概述

回合制战斗，玩家在地图上遇到怪物触发战斗，选择动作后服务器计算结果。

### 怪物

| 怪物 | 等级 | HP | ATK | DEF | SPD | 类型 | 掉落 |
|------|------|----|-----|-----|-----|------|------|
| 史莱姆 | 1 | 30 | 8 | 3 | 5 | 普通 | 草药、蘑菇 |
| 野狼 | 2 | 45 | 12 | 4 | 12 | 普通 | 狼皮、兽骨、肉干 |
| 毒蛛 | 3 | 35 | 10 | 2 | 14 | 普通 | 解毒剂、布料（毒液攻击） |
| 哥布林 | 3 | 50 | 14 | 6 | 10 | 普通 | 药水、绷带、铁矿（谨慎AI） |
| 暗熊 ★ | 5 | 100 | 20 | 12 | 6 | 精英 | 兽骨、力量药剂、皮甲 |

### 战斗动作

| 动作 | 效果 | 快捷键 |
|------|------|--------|
| 攻击 | 对怪物造成伤害，可能暴击 | 1 |
| 防御 | 本回合受到伤害减少 50% | 2 |
| 技能 | 消耗 MP 释放技能，打开技能选择面板 | 3 |
| 使用物品 | 消耗背包中的消耗品，打开物品选择面板 | 4 |
| 逃跑 | 基于速度的概率判定 | 5 |

### 伤害公式

```
基础伤害 = 攻击力 × 100 / (100 + 防御力)
浮动范围 = 基础伤害 × random(0.9, 1.1)
暴击率   = clamp(5%, 25%, 5% + 速度差 × 0.5%)
暴击伤害 = 浮动范围 × 1.5
防御减伤 = × 0.5（防御姿态时生效）
魔法伤害 = 攻击力 × power × 100 / (100 + 防御力 × 0.5)
最终伤害 = max(1, floor(计算结果))
```

### 逃跑公式

```
速度比 = 玩家速度 / 怪物速度
逃跑率 = clamp(10%, 90%, 30% + (速度比 - 1) × 30%)
```

### 状态效果

| 效果 | 类型 | 效果 |
|------|------|------|
| 中毒 | 持续伤害 | 每回合失去 5% 最大生命值 |
| 灼烧 | 持续伤害 | 每回合失去 3% 最大生命值 |
| 冻结 | 控制 | 无法行动 1 回合 |
| 攻击增强 | 增益 | 攻击力 +N 持续 N 回合 |
| 速度增强 | 增益 | 速度 +N 持续 N 回合 |
| 防御增强 | 增益 | 防御力 +N% 持续 N 回合 |

### 战斗奖励

- 击杀获得经验值和金币（金币在 [min, max] 范围随机）
- 掉落物品独立概率判定（每项单独 roll）
- 经验累积触发升级，升级提升全属性并回满 HP/MP
- 失败惩罚：HP 设为 1，扣除少量金币

### 怪物 AI

怪物行为由加权随机决定：

| 行为类型 | 说明 |
|----------|------|
| aggressive | 偏好攻击 |
| cautious | 低血量时增加防御概率 |
| defensive | 偏好防御 |

特殊技能（如毒蛛的毒液攻击）由 `ai.special` 配置定义。

### 物品效果定义

物品效果定义在 `backend/item_system.py` 的 `ITEM_EFFECTS` 中：

```python
ITEM_EFFECTS = {
    "health_potion": {"type": "heal", "value": 30},
    "mana_potion": {"type": "restore_mp", "value": 50},
    "strength_elixir": {"type": "buff", "stat": "attack", "value": 5, "duration": 3},
}
```

| 效果类型 | 参数 | 说明 |
|----------|------|------|
| heal | value | 恢复 HP |
| restore_mp | value | 恢复 MP |
| cure | effect | 解除状态 |
| buff | stat, value, duration | 临时增益 |
| learn_skill | skill_id | 学习技能 |

- 技能书在背包中使用，学会对应技能
- 使用后有使用反馈提示
