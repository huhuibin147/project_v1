# 游戏设计与优化文档

> 最后更新：2026-05-18
> 基于版本：怪物技能类型扩展完成后

## 概述

LLM NPC 像素风 RPG 是一个基于 Web 的像素风角色扮演游戏。玩家在村庄和森林中探索，与 LLM 驱动的 NPC 自由对话、交易物品、战斗怪物。

| 层级 | 技术 |
|------|------|
| 前端 | HTML5 Canvas + 原生 JavaScript |
| 后端 | Python FastAPI |
| LLM | OpenAI 兼容协议（支持 DeepSeek、通义千问、Ollama 等） |

## 系统总览

```
RPG 系统
├── 角色与成长 ── 属性 & 状态、职业、经验等级、技能、Buff/Debuff、天赋
├── 战斗 ────── 回合制战斗、伤害计算、怪物 & AI、多敌人、BOSS 战、属性克制
├── 物品与经济 ── 背包仓库、装备系统、消耗品、锻造、词条、商店 & 交易、掉落
├── 任务与剧情 ── 主线 & 支线、对话 & 选项、任务状态、剧情演出
├── 探索与地图 ── 地图场景、NPC 交互、传送点、天气昼夜、隐藏要素
└── 外围系统 ── 存档 & 设置、新手引导、成就 & 收集、NPC 日程
```

│                    fetch API                     │
├─────────────────────────────────────────────────┤
│                   后端 (FastAPI)                  │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
│  │ 地图  │ │ 玩家  │ │ NPC  │ │ 战斗  │ │ 物品  │  │

### 前端架构

- **渲染**：Canvas 逐像素绘制，无外部图片资源
- **管理器模块**：`GameManager`（核心）、`InputManager`（输入）、`RenderManager`（渲染优化）、`LoadingManager`（加载状态）
- **游戏循环**：`requestAnimationFrame`，每帧 `update(dt)` + `render()`
- **脏矩形优化**：只在状态变化时重绘，减少不必要的渲染
- **摄像机**：跟随玩家居中，边界钳制，视口裁剪优化
- **深度排序**：NPC、怪物、玩家按 Y 坐标排序绘制
- **UI 面板**：DOM 元素覆盖在 Canvas 上，`.active` 类控制显示
- **互斥机制**：所有面板共享 `combatOpen`/`dialogueOpen`/`inventoryOpen` 等标志

### 后端架构

- **模块化路由**：按功能拆分为 6 个路由模块（`routes/npc.py`、`routes/player.py`、`routes/map.py`、`routes/combat.py`、`routes/forge.py`、`routes/quest.py`）
- **单例玩家**：全局 `PlayerProfile` 实例，所有操作自动存档
- **懒加载 NPC**：首次访问时创建，存档切换时重建
- **内存战斗**：`CombatSession` 仅存内存，10 分钟过期自动清理
- **配置驱动**：物品、NPC、怪物、技能、地图均为 JSON 配置
- **共享上下文**：通过 `routes/context.py` 管理全局状态
- **统一异常处理**：请求验证错误和内部异常的统一返回格式

### 核心公式

```
基础伤害 = 攻击力 × 100 / (100 + 防御力)
暴击率   = clamp(5%, 25%, 5% + 速度差 × 0.5%)
逃跑率   = clamp(10%, 90%, 30% + (速度比 - 1) × 30%)
属性克制 = 火>草>水>火（克制 1.5x，被克制 0.67x，同属性 1.0x）
```

### 战斗动作

| 动作 | 快捷键 | 效果 |
|------|--------|------|
| 攻击 | 1 | 造成伤害，可能暴击 |
| 防御 | 2 | 减伤 50% |
| 技能 | 3 | 消耗 MP 释放技能 |
| 物品 | 4 | 使用消耗品 |
| 逃跑 | 5 | 概率逃跑 |

### 属性克制系统

```
🔥 火(fire) > 🌿 草(grass) > 💧 水(water) > 🔥 火(fire)
```

- 玩家元素：法师→火，盗贼→草，战士→水
- 怪物元素：根据怪物类型分配（植物系→草，法师系→火，野兽→无）
- 技能元素：火系技能覆盖为 fire，水系技能覆盖为 water
- 前端显示：怪物卡片显示元素图标，战斗日志显示克制/被克制提示

详见 → [combat_system.md](combat_system.md)

### 已完成功能

- ✅ 多敌人战斗：`CombatSession` 支持 1-3 个怪物 + BOSS 战
- ✅ 怪物组系统：地图配置支持 `monster_groups`，可配置多怪物编队
- ✅ BOSS 阶段系统：HP 阈值触发阶段切换，阶段可改变 AI 行为和属性
- ✅ BOSS 免疫机制：BOSS 免疫眩晕/冻结，精英 BOSS 额外免疫中毒
- ✅ AOE 技能：群体伤害按目标数量递减（1 目标 100% → 3 目标 65%）
- ✅ 目标选择：前端支持多怪物槽位，Tab 键切换目标，点击选择目标
- ✅ 状态效果互斥：灼烧和冻结互斥，施加一个会移除另一个
- ✅ 可堆叠效果：流血效果支持叠加（最多 3 层），每层增加伤害
- ✅ 属性克制：所有伤害计算点传入元素参数，前端显示克制提示

### 当前问题与优化方向

| 问题 | 优先级 | 说明 |

### 5.8 实时小地图

| 属性 | 值 |
|------|-----|
| 位置 | 右下角 HUD 区域 |
| 尺寸 | 160×120 像素（固定） |
| 缩放 | 自动适配当前地图尺寸 |
| 显示内容 | 瓦片缩略图 + 玩家（蓝色点）+ NPC（绿色点）+ 怪物（红色点）+ 传送门（紫色点） |
| 交互 | 无，纯展示 |

**渲染逻辑：**
- 每帧从 `currentMap.layers.ground` 读取瓦片数据
- 按 `mapWidth / minimapWidth` 和 `mapHeight / minimapHeight` 计算缩放比例
- 遍历瓦片时跳步采样（每 N 个瓦片取 1 个），用 `tileConfig` 中的颜色绘制
- 在缩略图上叠加玩家、NPC、怪物、传送门的像素点
- 玩家位置用蓝色闪烁点表示，周围有半透明圆形范围指示

**性能优化：**
- 小地图使用独立 Canvas 元素，避免与主游戏循环冲突
- 瓦片颜色缓存：只在地图切换时重新采样，移动时只更新动态点位置
- 跳步采样：50×40 地图 → 160×120 小地图，每 3-4 个瓦片采样 1 个
