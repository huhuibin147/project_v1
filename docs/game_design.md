# 游戏设计文档

## 概述

LLM NPC 像素风 RPG 是一个基于 Web 的像素风角色扮演游戏。玩家在村庄和森林中探索，与 LLM 驱动的 NPC 自由对话、交易物品、战斗怪物。

| 层级 | 技术 |
|------|------|
| 前端 | HTML5 Canvas + 原生 JavaScript |
| 后端 | Python FastAPI |
| LLM | OpenAI 兼容协议（支持 DeepSeek、通义千问、Ollama 等） |

## 系统总览

```
RPG 系统
├── 角色与成长 ── 属性 & 状态、职业、经验等级、技能、Buff/Debuff、天赋
├── 战斗 ────── 回合制战斗、伤害计算、怪物 & AI、多敌人、BOSS 战
├── 物品与经济 ── 背包仓库、装备系统、消耗品、锻造、词条、商店 & 交易、掉落
├── 任务与剧情 ── 主线 & 支线、对话 & 选项、任务状态、剧情演出
├── 探索与地图 ── 地图场景、NPC 交互、传送点、天气昼夜、隐藏要素
└── 外围系统 ── 存档 & 设置、新手引导、成就 & 收集、NPC 日程
```

### 前端架构

- **渲染**：Canvas 逐像素绘制，无外部图片资源
- **游戏循环**：`requestAnimationFrame`，每帧 `update(dt)` + `render()`
- **摄像机**：跟随玩家居中，边界钳制，视口裁剪优化
- **深度排序**：NPC、怪物、玩家按 Y 坐标排序绘制
- **UI 面板**：DOM 元素覆盖在 Canvas 上，`.active` 类控制显示
- **互斥机制**：所有面板共享 `combatOpen`/`dialogueOpen`/`inventoryOpen` 等标志

### 后端架构

- **单例玩家**：全局 `PlayerProfile` 实例，所有操作自动存档
- **懒加载 NPC**：首次访问时创建，存档切换时重建
- **内存战斗**：`CombatSession` 仅存内存，10 分钟过期自动清理
- **配置驱动**：物品、NPC、怪物、技能、地图均为 JSON 配置

### 核心公式

```
基础伤害 = 攻击力 × 100 / (100 + 防御力)
暴击率   = clamp(5%, 25%, 5% + 速度差 × 0.5%)
逃跑率   = clamp(10%, 90%, 30% + (速度比 - 1) × 30%)
```

### 战斗动作

| 动作 | 快捷键 | 效果 |
|------|--------|------|
| 攻击 | 1 | 造成伤害，可能暴击 |
| 防御 | 2 | 减伤 50% |
| 技能 | 3 | 消耗 MP 释放技能 |
| 物品 | 4 | 使用消耗品 |
| 逃跑 | 5 | 概率逃跑 |

### 5.8 实时小地图

| 属性 | 值 |
|------|-----|
| 位置 | 右下角 HUD 区域 |
| 尺寸 | 160×120 像素（固定） |
| 缩放 | 自动适配当前地图尺寸 |
| 显示内容 | 瓦片缩略图 + 玩家（蓝色点）+ NPC（绿色点）+ 怪物（红色点）+ 传送门（紫色点） |
| 交互 | 无，纯展示 |

**渲染逻辑：**
- 每帧从 `currentMap.layers.ground` 读取瓦片数据
- 按 `mapWidth / minimapWidth` 和 `mapHeight / minimapHeight` 计算缩放比例
- 遍历瓦片时跳步采样（每 N 个瓦片取 1 个），用 `tileConfig` 中的颜色绘制
- 在缩略图上叠加玩家、NPC、怪物、传送门的像素点
- 玩家位置用蓝色闪烁点表示，周围有半透明圆形范围指示

**性能优化：**
- 小地图使用独立 Canvas 元素，避免与主游戏循环冲突
- 瓦片颜色缓存：只在地图切换时重新采样，移动时只更新动态点位置
- 跳步采样：50×40 地图 → 160×120 小地图，每 3-4 个瓦片采样 1 个

### 5.9 菜单大地图（M 键）

| 属性 | 值 |
|------|-----|
| 打开方式 | 按 M 键或菜单面板点击「查看地图」按钮 |
| 尺寸 | 600×450 像素（居中面板） |
| 显示内容 | 当前地图完整缩略图 + 已探索区域高亮 + 传送门连线 |
| 交互 | 鼠标悬停显示坐标信息，点击传送门可快速传送（需已探索） |

**渲染逻辑：**
- 使用独立 Canvas 绘制完整地图缩略图
- 已探索区域：玩家曾经到达过的瓦片区域，用半透明遮罩标记
- 传送门连线：用虚线连接所有传送门及其目标位置
- 玩家位置用大蓝色箭头标记，带脉冲动画
- NPC 和怪物用小圆点标记（可选开关）

**探索记录：**
- 后端 `player_profile.py` 新增 `explored_tiles` 字段，记录玩家到过的瓦片坐标
- 前端 `player.js` 在玩家移动时定期上报探索位置
- 大地图面板根据 `explored_tiles` 渲染已探索/未探索区域

**快捷键：**
- M 键：打开/关闭大地图
- ESC 键：关闭大地图
- 大地图打开时禁止玩家移动

### 5.10 采集系统

| 属性 | 值 |
|------|-----|
| 交互方式 | 靠近采集点（1格内）按 E 键 |
| 冷却时间 | 60-90秒（根据物品类型） |
| 物品类型 | 草药、蘑菇、铁矿石、兽骨 |
| 视觉反馈 | 不同颜色标识 + 冷却灰色 + 可采集闪烁 |
| 提示系统 | 靠近时显示"按 E 采集 XXX" |

**采集机制：**
- 采集点分布在所有地图的可行走格子上
- 采集成功后物品自动放入背包
- 采集点进入冷却状态，显示为灰色
- 冷却结束后恢复原色并闪烁提示
- 采集进度自动计入任务系统

**采集点分布：**
- 青石村：蘑菇×2、草药×1
- 幽暗森林：蘑菇×3、兽骨×2、草药×2
- 黑暗洞穴：铁矿石×4、蘑菇×2
