# 游戏设计文档

## 概述

LLM NPC 像素风 RPG 是一个基于 Web 的像素风角色扮演游戏。玩家在村庄和森林中探索，与 LLM 驱动的 NPC 自由对话、交易物品、战斗怪物。

| 层级 | 技术 |
|------|------|
| 前端 | HTML5 Canvas + 原生 JavaScript |
| 后端 | Python FastAPI |
| LLM | OpenAI 兼容协议（支持 DeepSeek、通义千问、Ollama 等） |

## 架构设计

```
┌─────────────────────────────────────────────────┐
│                   前端 (Canvas + DOM)             │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
│  │ 地图  │ │ 玩家  │ │ NPC  │ │ 战斗  │ │ UI   │  │
│  └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘  │
│     └────────┴────────┴────────┴────────┘       │
│                    fetch API                     │
├─────────────────────────────────────────────────┤
│                   后端 (FastAPI)                  │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
│  │ 地图  │ │ 玩家  │ │ NPC  │ │ 战斗  │ │ 物品  │  │
│  └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘ └──┬───┘  │
│     └────────┴────────┴────────┴────────┘       │
│              JSON 文件持久化 (data/)              │
├─────────────────────────────────────────────────┤
│              LLM API (OpenAI 兼容)               │
└─────────────────────────────────────────────────┘
```

### 前端架构

- **渲染**：Canvas 逐像素绘制，无外部图片资源
- **游戏循环**：`requestAnimationFrame`，每帧 `update(dt)` + `render()`
- **摄像机**：跟随玩家居中，边界钳制，视口裁剪优化
- **深度排序**：NPC、怪物、玩家按 Y 坐标排序绘制，实现遮挡
- **UI 面板**：DOM 元素覆盖在 Canvas 上，`.active` 类控制显示
- **互斥机制**：所有面板共享 `combatOpen`/`dialogueOpen`/`inventoryOpen` 等标志

### 后端架构

- **单例玩家**：全局 `PlayerProfile` 实例，所有操作自动存档
- **懒加载 NPC**：首次访问时创建，存档切换时重建
- **内存战斗**：`CombatSession` 仅存内存，10 分钟过期自动清理
- **配置驱动**：物品、NPC、怪物、地图均为 JSON 配置

## 玩家系统

### 职业

| 职业 | HP | 攻击 | 防御 | 速度 | 定位 |
|------|-----|------|------|------|------|
| 战士 | 120 | 15 | 12 | 8 | 高血量高防御 |
| 盗贼 | 80 | 18 | 6 | 15 | 高攻高速 |
| 法师 | 70 | 22 | 5 | 10 | 高攻低防 |

### 属性计算

```
基础属性 = 职业基础 + (等级-1) × 成长值
总属性   = 基础属性 + 装备加成

成长值：HP +10, 攻击 +3, 防御 +2, 速度 +1（每级）
```

### 等级系统

- 经验值累积触发升级
- 升级所需经验递增（×1.5）
- 升级后自动重新计算属性
- 经验来源：战斗胜利、任务完成（预留）

### 存档系统

- 3 个存档槽，完全隔离
- 存档结构：`data/save_{slot}/player.json` + `{npc_id}.json`
- 游戏每 5 秒自动保存位置
- 切换存档时自动清空并重新加载所有数据

## NPC 系统

### 当前 NPC

| NPC | 身份 | 位置 | 商品类型 |
|-----|------|------|----------|
| 铁匠老王 | 武器商人 | 村庄铁匠铺 | 武器、盾牌、防具、药水、工具 |
| 杂货婆刘婶 | 日用品商人 | 村庄杂货铺 | 食物、布料、绷带、日用品 |
| 采药人老林 | 药草商人 | 森林深处 | 药水、饰品、草药、魔法水晶 |

### LLM 驱动对话

NPC 由大语言模型驱动，每次对话构建包含以下上下文的 Prompt：

```
System: 角色设定 + 性格参数 + 当前情绪/好感度 + 商店库存 + 玩家背包 + 对话历史（最近10轮）
User: 玩家输入
→ LLM 返回结构化 JSON
```

**LLM 输出格式**：
```json
{
  "reply": "NPC 对话（1-3句）",
  "intent": "chat|quest|trade|unknown",
  "mood": "当前情绪",
  "affinity_change": -10到+10,
  "trade_action": null | {"action": "buy|sell", "item_id": "...", "quantity": N}
}
```

### 性格系统

每个 NPC 有 4 个性格参数（0-1），影响 LLM 输出风格：

| 参数 | 说明 |
|------|------|
| friendliness | 友善度 |
| courage | 勇气 |
| greed | 贪婪度 |
| humor | 幽默感 |

### 好感度与情绪

- 好感度范围 0-100，初始值由 NPC 配置定义
- 玩家言行（夸赞、侮辱、帮助）影响好感度变化
- 情绪随对话动态变化（平静/高兴/担忧/生气）
- 好感度影响 NPC 态度和交易行为

### 意图识别

LLM 自动分类玩家意图：

| 意图 | 触发条件 | 行为 |
|------|----------|------|
| chat | 日常对话 | NPC 自由回复 |
| quest | 询问任务 | NPC 描述任务（预留） |
| trade | 明确说"买"/"卖" | 自动执行交易 |
| unknown | 无法识别 | NPC 礼貌回应 |

### 交易系统

**两种交易路径**：
1. **对话驱动**：玩家在对话中表达交易意图，LLM 识别后自动执行
2. **商店面板**：按 E 打开 NPC 交互选项，选择"商店"直接交易

**交易流程**：
- 购买：校验玩家金币 → 扣除金币 → 物品加入背包
- 出售：校验玩家物品 → 扣除物品 → 金币加入背包
- NPC 收购的物品存入独立的收购仓库，不混入商店库存

## 地图系统

### 当前地图

| 地图 | 名称 | 尺寸 | 说明 |
|------|------|------|------|
| village | 青石村 | 50×40 | 起始村庄，安全区 |
| forest | 幽暗森林 | 60×50 | 森林区域，有怪物 |

### 地图数据格式

```json
{
  "id": "map_id",
  "name": "显示名称",
  "width": 50,
  "height": 40,
  "tile_size": 32,
  "layers": { "ground": [[tile_id, ...], ...] },
  "monsters": [{ "monster_id": "...", "x": N, "y": N, "patrol": [...] }],
  "objects": [{ "id": "...", "type": "...", "x": N, "y": N, "properties": {...} }],
  "npcs": [{ "npc_id": "...", "x": N, "y": N }],
  "player_spawn": { "x": N, "y": N }
}
```

### 瓦片系统

瓦片定义在 `config/tiles.json`，每种瓦片有：名称、颜色、是否可行走、渲染细节标识。

当前 15 种瓦片：草地、泥土路、砖墙、屋顶、树木、水面、木地板、栅栏、石头、石板路、沙地、雪地、洞穴墙、洞穴地板等。

### 交互物件

| 类型 | 说明 | 交互方式 |
|------|------|----------|
| portal | 传送门 | 踩上去自动触发 |
| chest | 宝箱 | 按 E 打开（一次性） |
| gather | 采集点 | 按 E 采集（有冷却） |
| decoration | 装饰物 | 按 E 查看文字 |

### 怪物刷新

地图 JSON 中的 `monsters` 数组定义怪物刷新点：

```json
{
  "monster_id": "wild_wolf",
  "x": 40, "y": 20,
  "patrol": [
    {"x": 38, "y": 20},
    {"x": 42, "y": 20},
    {"x": 42, "y": 24}
  ]
}
```

- `patrol` 可选，有则怪物在路径点间巡逻，无则原地站立
- 村庄地图 `monsters: []`（安全区）

## 战斗系统

### 概述

回合制战斗，玩家在地图上遇到怪物触发战斗，选择动作后服务器计算结果。

### 怪物

| 怪物 | 等级 | HP | ATK | DEF | SPD | 类型 | 掉落 |
|------|------|----|-----|-----|-----|------|------|
| 史莱姆 | 1 | 30 | 8 | 3 | 5 | 普通 | 草药、蘑菇 |
| 野狼 | 2 | 45 | 12 | 4 | 12 | 普通 | 狼皮、兽骨、肉干 |
| 毒蛛 | 3 | 35 | 10 | 2 | 14 | 普通 | 解毒剂、布料（毒液攻击） |
| 哥布林 | 3 | 50 | 14 | 6 | 10 | 普通 | 药水、绷带、铁矿（谨慎AI） |
| 暗熊 ★ | 5 | 100 | 20 | 12 | 6 | 精英 | 兽骨、力量药剂、皮甲 |

### 战斗动作

| 动作 | 效果 |
|------|------|
| 攻击 | 对怪物造成伤害，可能暴击 |
| 防御 | 本回合受到伤害减少 50% |
| 使用物品 | 消耗背包中的消耗品 |
| 逃跑 | 基于速度的概率判定 |

### 伤害公式

```
基础伤害 = 攻击力 × 100 / (100 + 防御力)
浮动范围 = 基础伤害 × random(0.9, 1.1)
暴击率   = clamp(5%, 25%, 5% + 速度差 × 0.5%)
暴击伤害 = 浮动范围 × 1.5
防御减伤 = × 0.5（防御姿态时生效）
最终伤害 = max(1, floor(计算结果))
```

### 逃跑公式

```
速度比 = 玩家速度 / 怪物速度
逃跑率 = clamp(10%, 90%, 30% + (速度比 - 1) × 30%)
```

### 状态效果

| 效果 | 类型 | 效果 |
|------|------|------|
| 中毒 | 持续伤害 | 每回合失去 5% 最大生命值 |
| 灼烧 | 持续伤害 | 每回合失去 3% 最大生命值 |
| 攻击增强 | 增益 | 攻击力 +N 持续 N 回合 |
| 速度增强 | 增益 | 速度 +N 持续 N 回合 |

### 战斗奖励

- 击杀获得经验值和金币（金币在 [min, max] 范围随机）
- 掉落物品独立概率判定（每项单独 roll）
- 经验累积触发升级，升级提升全属性
- 失败惩罚：HP 设为 1，扣除少量金币

### 怪物 AI

怪物行为由加权随机决定：

| 行为类型 | 说明 |
|----------|------|
| aggressive | 偏好攻击 |
| cautious | 低血量时增加防御概率 |
| defensive | 偏好防御 |

特殊技能（如毒蛛的毒液攻击）由 `ai.special` 配置定义。
