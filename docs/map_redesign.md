## 一、现状分析

### 1.1 当前地图问题

| 问题 | 说明 | 影响 |
|------|------|------|
| 布局过于简单 | 村庄仅十字路口+几栋房子，森林是随机散布的树木 | 缺乏探索感，玩家一眼看穿 |
| 缺乏区域划分 | 洞穴/沙漠虽有 level_range 但地图内无视觉区分 | 玩家无法感知"深入" |
| 建筑千篇一律 | 所有建筑都是 4(墙)+14(木地板) 的方盒子 | 无辨识度，无地标感 |
| 道路系统单调 | 只有直线十字路，无蜿蜒小径/环形路 | 缺乏引导性和发现感 |
| 无水系/地形变化 | 除森林有一条直线河外，其余地图无水系 | 地图缺乏自然感 |
| 生成器能力弱 | 只能生成简单十字路+随机散布 | 无法生成有设计感的地图 |
| 无子区域概念 | 地图是单一平面，无"区"的概念 | 难以做渐进难度和叙事 |

### 1.2 经典游戏地图设计参考

**《星露谷物语》— 村庄设计**：
- 中心广场（地标）→ 放射状道路连接各功能区
- 功能分区明确：住宅区、商业区、农场区、自然区
- 水系贯穿（河流穿过村庄，桥连接两岸）
- 每栋建筑有独特造型和门前装饰

**《塞尔达》— 森林/洞穴设计**：
- 迷宫式路径：主干道 + 隐藏岔路
- 渐进深度：入口区（安全）→ 深处（危险）→ 核心区（BOSS）
- 地标引导：火把、光源标记路径
- 环境叙事：越深越暗，装饰从生机→衰败

**《暗黑破坏神》— 洞穴/地下城设计**：
- 房间+走廊模式：开放房间由窄道连接
- 随机生成但保证可达性
- 小地图/迷雾系统增加探索感

**《原神》— 城镇设计**：
- 多层高差：台阶、平台、桥梁
- 功能分区+视觉地标（教堂、塔楼、市场）
- 环境叙事：NPC 位置与区域功能匹配

## 二、设计目标

1. **区域化**：每张地图划分为 2-4 个子区域，各有主题和难度
2. **地标化**：每区域至少一个视觉地标（广场、塔楼、湖泊、遗迹）
3. **路径多样化**：主干道 + 支路 + 隐藏路径，非直线十字
4. **自然地形**：水系蜿蜒、地形起伏、植被分层
5. **建筑差异化**：不同功能建筑有不同瓦片组合和门前装饰
6. **生成器升级**：支持区域化、地标化、路径多样化的地图生成
7. **可扩展**：新增地图只需编写区域配置，无需改代码

## 三、核心概念设计

### 3.1 区域系统（Zone）

每张地图由多个区域组成，区域定义了该区域的主题、难度和内容。

```json
{
  "zones": [
    {
      "id": "village_center",
      "name": "村中心广场",
      "bounds": {"x": 18, "y": 15, "w": 14, "h": 12},
      "theme": "plaza",
      "level_range": [1, 1],
      "landmarks": ["fountain"],
      "ground_tile": 22,
      "road_tile": 1
    },
    {
      "id": "village_residential",
      "name": "居民区",
      "bounds": {"x": 3, "y": 3, "w": 14, "h": 12},
      "theme": "residential",
      "level_range": [1, 1],
      "landmarks": [],
      "ground_tile": 0,
      "road_tile": 1
    }
  ]
}
```

区域属性说明：
- `bounds`：区域在地图上的矩形范围
- `theme`：区域主题，决定建筑类型和装饰风格
- `landmarks`：区域内的地标建筑列表
- `ground_tile`/`road_tile`：该区域使用的地面/道路瓦片

### 3.2 地标系统（Landmark）

地标是区域的核心视觉焦点，由预定义的瓦片模板组成。

```json
{
  "landmarks": {
    "fountain": {
      "name": "喷泉广场",
      "width": 5, "height": 5,
      "tiles": [
        [22, 22, 22, 22, 22],
        [22,  5,  5,  5, 22],
        [22,  5, 16,  5, 22],
        [22,  5,  5,  5, 22],
        [22, 22,  1, 22, 22]
      ],
      "door": [2, 4]
    },
    "watchtower": {
      "name": "瞭望塔",
      "width": 3, "height": 5,
      "tiles": [
        [4, 4, 4],
        [4, 22, 4],
        [4, 22, 4],
        [4, 22, 4],
        [4, 4, 1]
      ],
      "door": [1, 4]
    }
  }
}
```

### 3.3 路径系统（Path）

路径不再是简单的直线，而是支持多种类型：

```json
{
  "paths": [
    {
      "type": "main_road",
      "points": [[25, 2], [25, 15], [25, 20], [25, 38]],
      "width": 2,
      "tile": 1
    },
    {
      "type": "winding_path",
      "points": [[5, 20], [10, 18], [15, 22], [18, 20]],
      "width": 1,
      "tile": 1
    },
    {
      "type": "hidden_path",
      "points": [[40, 10], [42, 12], [44, 15]],
      "width": 1,
      "tile": 0,
      "hidden": true
    }
  ]
}
```

路径类型：
- `main_road`：主干道，宽 2 格，明显可见
- `winding_path`：蜿蜒小径，宽 1 格，连接区域
- `hidden_path`：隐藏路径，使用草地瓦片，需探索发现

### 3.4 建筑模板系统

建筑不再是简单的方盒子，而是有门廊、装饰、功能区分的模板：

```json
{
  "building_templates": {
    "house_with_yard": {
      "name": "带院子的民居",
      "width": 9, "height": 8,
      "tiles": [
        [0,  0,  8,  8,  8,  8,  8,  0,  0],

        [4, 14, 14, 14, 14, 14, 14, 14,  4],
        [4,  4,  4,  1,  4,  4,  4,  4,  4]
      ],
      "door": [3, 7],
      "yard_entrance": [3, 2]
    },
    "shop_with_sign": {
      "name": "带招牌的商店",
      "width": 7, "height": 6,
      "tiles": [
        [4,  4,  4,  6,  4,  4,  4],
```

水系不再是直线，而是自然蜿蜒的河流和湖泊：

## 四、各地图改版方案

### 4.1 青石村（village）— 50×40 → 60×50

**设计理念**：参考星露谷，打造有生活气息的起始村庄

**区域划分**：
```
┌──────────────────────────────────────────────────────┐
│  北门区    │  居民区（左）  │  居民区（右）  │  东门区  │
│  传送门    │  民居×3+院子  │  民居×2+花园  │  传送门  │
├────────────┼────────────────┼────────────────┤          │
│            │     村中心广场     │            │          │
│  农田区    │   喷泉+公告板    │  商业区     │          │
│  田地×4   │     十字路口      │  商店+铁匠  │          │
│            │                  │  酒馆+学院  │          │
├────────────┼────────────────┼────────────────┤          │
│  西门区    │   神殿区        │  河岸区      │          │
│  传送门    │   神殿+墓地     │  河流+桥    │          │
└──────────────────────────────────────────────────────┘
```

**关键改进**：
1. 中心广场：石板地面 + 喷泉地标 + 公告板
2. 蜿蜒河流：从西北流向东南，2 座桥连接两岸
3. 带院子的民居：栅栏围合 + 花园/菜地
4. 农田区：4 块田地（泥土路分隔），增加村庄生活感
5. 神殿区：独立区域，石板路连接，有墓地
6. 四个方向传送门：北→森林、东→王城、西→沙漠、南→洞穴

**新增瓦片需求**：
- 25: 田地（棕色+绿色条纹）
- 26: 墓碑（深灰+十字）
- 27: 公告板（木色+白色）

4. 遗迹地标：石柱围成的圆形广场，BOSS 刷新点
5. 环境叙事：浅林区有花丛和蝴蝶，深林区有枯树和兽骨

## 五、地图生成器升级方案

### 5.1 新增区域化生成流程

```
1. 读取区域配置 → 为每个区域填充基础瓦片
2. 放置地标建筑 → 在区域中心放置地标
3. 生成路径网络 → 连接区域中心点（主干道+支路）
4. 生成水系     → 按配置生成河流/湖泊
5. 放置建筑     → 按区域主题放置建筑模板
6. 添加装饰     → 按区域主题添加植被/石头/花丛
7. 放置交互物件 → 宝箱/采集点/传送门
8. 验证可达性   → BFS 确保所有重要位置可达
```

### 5.2 区域配置文件

新增 `config/map_templates/` 目录，每个地图一个模板文件：

```
config/map_templates/
├── village.json      # 青石村模板
├── forest.json       # 幽暗森林模板
├── dark_cave.json    # 幽暗洞穴模板
├── desert_oasis.json # 沙漠绿洲模板
└── royal_city.json   # 王城模板
```

模板文件包含：区域定义、地标位置、路径点、水系配置、建筑布局、装饰规则。
