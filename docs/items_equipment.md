# 物品与装备系统

物品系统提供经济循环（金币、背包、交易），装备系统在此基础上让物品可穿戴并影响角色属性。

## 物品数据结构

物品定义在 `config/items.json`，全局共享。

| 字段 | 说明 |
|------|------|
| `id` | 唯一标识，如 `iron_sword` |
| `name` | 显示名称 |
| `type` | weapon / armor / consumable / food / tool / material / accessory |
| `description` | 物品描述 |
| `buy_price` | 商店售价（0 = 不可购买） |
| `sell_price` | 收购价（0 = 不可出售） |
| `stackable` | 是否可叠加 |
| `equip_slot` | 装备槽位（仅装备类物品） |
| `stats` | 属性加成（仅装备类物品） |
| `rarity` | 稀有度等级，默认 common |
| `tier` | 等级段：tier1 / tier2 / tier3 |
| `level_range` | 使用等级上限 |
| `affixes` | 特殊词条列表（预留） |
| `forge_info` | 锻造信息（预留） |

## 装备槽位

| 槽位 | key | 可装备类型 |
|------|-----|-----------|
| 武器 | `weapon` | weapon |
| 盾牌 | `shield` | armor（shield 子类） |
| 头部 | `head` | armor（head 子类） |
| 身体 | `body` | armor（body 子类） |
| 饰品 | `accessory` | accessory |

属性计算：`实际属性 = 职业基础 + 等级成长 + 装备加成总和`

## 交易系统

### 对话驱动交易

玩家在对话中表达交易意图，LLM 自动识别并执行。系统区分"买"和"卖"的明确意图，询问价格等不会误触发交易。NPC 知道玩家背包内容，可避免交易不存在的物品。

### 商店面板交易

靠近 NPC 按 E → 选择"商店" → 点击购买/出售按钮直接交易。

### 交易校验

- 购买：金币充足 + NPC 库存足够 + 物品可买
- 出售：玩家持有物品 + 物品可卖 + NPC 金币充足

NPC 收购的物品存入独立的收购仓库，不混入商店库存。

## 装备对比

背包和商店中的可装备物品显示与当前装备的属性差异：绿色 `+N` 表示提升，红色 `-N` 表示下降。

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/inventory` | GET | 玩家背包与金币 |
| `/api/shop?npc_id=xxx` | GET | NPC 商店库存 |
| `/api/trade` | POST | 直接交易 |
| `/api/equipment` | GET | 装备详情（含基础/加成/总计属性） |
| `/api/equip` | POST | 装备物品 |
| `/api/unequip` | POST | 卸下装备 |
| `/api/forge/recipes` | GET | 锻造配方（预留） |
| `/api/forge/craft` | POST | 执行锻造（预留） |
| `/api/affixes/types` | GET | 词条类型（预留） |
| `/api/affixes/enchant` | POST | 附魔词条（预留） |

## 预留系统

### 锻造

玩家收集材料后在铁匠铺锻造高级装备。物品 `forge_info` 字段预留配方结构：材料列表、产出物、费用、等级要求。

### 特殊词条

装备可附带词条提供额外效果。词条类型：攻击触发（灼烧、冰冻）、被动加成（暴击、闪避）、受击触发（反伤）、击杀触发（吸血）、条件触发（低血量增益）。物品 `affixes` 字段预留词条结构。
