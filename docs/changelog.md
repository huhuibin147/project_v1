# Changelog

> 项目变更记录，由 todolist 中已完成项归档而来。

---

## 2026-05-20

### 优化
- AOE 伤害改为统一乘数（65%），不再随目标数变化
- 雷霆一击 power 从 2.0 降至 1.6，并修正文档（仅限战士/法师）
- 战斗胜利/失败结果面板添加「查看记录」按钮，可返回查看战斗日志后再关闭
- 雷霆一击改为法师专属技能，取消战士共享
- 地图生成器优化：TILES字典动态加载tiles.json、建筑模板瓦片ID对齐、生成顺序修正(先建筑后道路)、道路碰撞检测、装饰物智能放置(最小间距+避让)、门口道路延伸
- 逃跑概率提升：基础概率30%→50%，速度比系数30%→40%，下限10%→20%，上限90%→95%
- 村庄地图：长河流分隔改为两个小池塘，避免地图被河流割裂

### Bug 修复
- NPC缺失：地图生成后 npcs 为空，已添加 NPC 缺失检测（交叉引用 npcs.json），自动在出生点附近放置缺失 NPC
- 传送门缺失：village 缺少到 royal_city 的传送门，已添加 portal_royal_city
- 怪物消失：地图模板生成时 monster_groups 为空，已添加 _auto_place_monsters 根据区域自动放置怪物
- 传送点卡死：desert_oasis→royal_city 传送门落点(5,37)在墙上，已修复为(5,39)
- 地图绕路：添加路径效率检测（BFS实际距离 vs 曼哈顿距离），效率低于40%时自动修路
- P键传送玩家：openPlayerInfo 中 fetchPlayerInfo 调用 setPlayerPosition 重置玩家位置，已移除该调用
- 技能菜单 updatePlayerInfo 未定义：升级技能时调用不存在的函数导致JS报错，已改为 Object.assign(playerInfo, ...)
- 技能菜单互斥逻辑矛盾：可自动关闭的面板同时出现在返回条件和关闭代码中，已修正
- 技能菜单 level_scaling 数据泄露：get_skill_at_level(level=1) 未移除内部字段，已修复
- 技能菜单 Tab 事件重复绑定：每次打开面板重新绑定 click 事件，已添加初始化标志
- 天赋面板互斥逻辑：同步修复与技能菜单相同的互斥问题

### 新功能
- 技能菜单界面：快捷键 K 打开技能菜单面板，支持已学/可学技能标签切换
- 技能升级系统：已学技能可花费金币升级（最高3级），升级后属性自动在战斗中生效
- 技能等级数据：skills.json 所有18个技能添加 max_level 和 level_scaling 字段
- 技能 API：GET /api/skills 获取技能信息、POST /api/skills/upgrade 升级技能
- 角色信息面板技能标签增强：显示技能等级 Lv.x/x
- 游戏菜单和帮助面板添加技能系统说明
