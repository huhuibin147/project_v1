## 一、现有问题分析

### 1.1 布局问题

| 问题 | 说明 |
|------|------|
| 怪物区域信息稀疏 | 怪物卡片只有 64×64 像素精灵 + 名称 + HP 条，缺少等级、类型标识、属性标识 |
| 玩家区域过于简陋 | 只有两个血条（HP/MP）+ 状态效果，没有玩家头像、属性概览、护盾显示 |
| 战斗面板宽度固定 520px | 多怪物时卡片挤在一起，信息密度低 |
| Tab 提示占据整行 | `#combat-target-hint` 独占操作栏上方一整行，多怪物时浪费空间 |
| 战斗日志与主区域混排 | 日志在怪物区和玩家区之间，视觉层次不清晰 |
| 技能/物品面板浮层定位 | 绝对定位浮层容易被遮挡，且与操作按钮视觉割裂 |

### 1.2 信息展示问题

| 问题 | 说明 |
|------|------|
| 怪物无等级显示 | 无法判断怪物强弱 |
| 怪物无属性标识 | 属性克制系统已有但 UI 无展示 |
| 玩家无护盾条 | `player_shield` 后端已有但前端无展示 |
| 状态效果只有英文缩写 | `poison(3)` 对玩家不友好，缺少图标和中文名 |
| 无怪物行动预览 | 无法预判怪物下回合行为 |
| 无回合数/战斗时长 | 只有回合数文字，缺少视觉进度感 |

### 1.3 交互问题

| 问题 | 说明 |
|------|------|
| 操作按钮快捷键提示占空间 | "攻击 (1)" 中括号内容冗余 |
| 多怪物目标切换不直观 | 只能 Tab 循环或点击小卡片 |
| 技能面板关闭需点 × | 无 ESC 键关闭的视觉提示 |
| 战斗结果面板信息少 | 只有经验/金币/掉落，无战斗统计 |

## 三、详细设计

### 3.1 整体布局

**面板尺寸**：从 520px 扩展到 640px，高度自适应（max-height: 92vh）

**分区结构**（从上到下）：

1. **顶栏**（48px）：回合数 + 战斗标题 + 怪物数量指示
2. **怪物区**（弹性高度，约 40%）：多怪物卡片横排
3. **信息区**（弹性高度，约 35%）：左侧玩家状态 + 右侧战斗日志
4. **操作栏**（56px）：5 个操作按钮

### 3.2 怪物卡片重设计

**现有**：64×64 精灵 + 名称 + HP 条，min-width: 120px

**新设计**：

```
┌─────────────────────┐
│  ★ 暗熊              │  ← 名称 + 精英/BOSS 标记
│  Lv.5 🌿森林 🛡防御  │  ← 等级 + 标签 + 行动意图
│  ┌─────────────────┐ │
│  │   64×64 精灵     │ │  ← 精灵区域（带待机呼吸动画）
│  └─────────────────┘ │
│  HP ████████░░ 80/100│  ← HP 条 + 数值
│  ☠️中毒(2) 🔥灼烧(1) │  ← 状态效果（图标+中文名+回合数）
│       ▶ 目标         │  ← 选中指示器
└─────────────────────┘
```

**尺寸**：min-width 从 120px → 160px，精灵保持 64×64

**新增信息**：
- 等级显示（`Lv.X`）
- 怪物类型标记（精英 ★、BOSS ♛）
- 状态效果中文化 + 图标
- 怪物行动意图图标（⚔攻击 / 🛡防御 / ✨技能）
- 选中指示器改为底部箭头 ▲ 而非右上角标签

### 3.3 玩家状态栏重设计

**现有**：HP 条 + MP 条 + 状态效果文字

**新设计**：

```
┌────────────────────────┐
│  ┌────┐                │
│  │头像 │  勇者  Lv.8    │  ← 玩家头像 + 名称 + 等级
│  │区域 │                │
│  └────┘                │
│  HP ██████████░ 112/120│  ← HP 条（绿→红渐变）
│  MP █████░░░░░  18/30  │  ← MP 条（蓝色）
│  🛡 ████████░░  40     │  ← 护盾条（金色，有护盾时显示）
│  效果: 🟢攻击↑ 🛡护盾   │  ← 状态效果（图标+中文名）
└────────────────────────┘
```

**新增信息**：
- 玩家头像区域（48×48 像素风格头像）
- 等级显示
- 护盾条（`player_shield` 后端已有数据）
- HP 条颜色随比例变化（>50% 绿色，25-50% 黄色，<25% 红色）
- 状态效果中文化 + 彩色图标

### 3.4 战斗日志优化

**现有**：min-height: 100px, max-height: 160px

**新设计**：
- 与玩家状态栏并排显示（右侧），占据约 60% 宽度
- max-height 增加到 180px
- 日志条目增加类型图标前缀：
  - ⚔ 玩家攻击
  - 🗡 怪物攻击
  - ✨ 技能
  - 🧪 物品
  - ☠️ 状态效果
  - 🏆 胜利
  - 💀 失败

### 3.5 操作栏重设计

**现有**：5 个等宽按钮 + Tab 提示占一整行

**新设计**：

```
┌──────────────────────────────────────────────┐
│  ⚔攻击  🛡防御  ✨技能  🎒物品  │  🏃逃跑  │
└──────────────────────────────────────────────┘
```

- 移除 `#combat-target-hint` 独占行，改为怪物卡片上的提示
- 按钮文字去掉快捷键提示（括号内容），快捷键改为 tooltip
- 逃跑按钮右对齐，视觉上与战斗操作区分
- 按钮增加图标前缀

### 3.6 技能/物品面板优化

**现有**：绝对定位浮层，320px 宽

**新设计**：
- 改为在操作栏位置展开（替换操作按钮区域）
- 宽度与面板等宽
- 技能行增加伤害类型标签（物理/魔法）、AOE 标签
- 物品行增加效果描述

### 3.7 战斗结果面板增强

**现有**：经验 + 金币 + 掉落物品

**新设计**：

```
┌──────────────────────────┐
│     🏆 战斗胜利！         │
│                          │
│  ── 战斗统计 ──          │
│  总回合数: 5             │
│  总伤害: 247             │
│  最大单次伤害: 68 (暴击)  │
│  暴击次数: 2             │
│                          │
│  ── 获得奖励 ──          │
│  ✨ 经验值: +80          │
│  💰 金币: +45            │
│  📦 掉落:                │
│    野兽之骨 x1           │
│    狼皮 x1               │
│                          │
│  🎉 等级提升！           │
│                          │
│      [ 继续 ]            │
└──────────────────────────┘
```

## 六、实现计划

### Phase 1：HTML 结构重构 ✅
- 重写 `#combat-panel` 内部 HTML 结构
- 新增玩家头像区域、护盾条、怪物等级/意图等元素

### Phase 2：CSS 样式重写 ✅
- 面板宽度 520px → 640px
- 怪物卡片 120px → 160px，新增等级/意图/效果样式
- 玩家区域改为左侧卡片式布局
- 日志区与玩家区并排
- 操作栏移除 Tab 提示行
- 技能/物品面板改为内嵌展开式

### Phase 3：JS 逻辑适配 ✅
- `renderCombat()` 适配新 HTML 结构
- `renderMonsterSlots()` 增加等级、意图、效果中文化
- `renderCombat()` 增加护盾条、HP 颜色渐变
- `appendCombatLog()` 增加类型图标前缀
- `showCombatResult()` 增加战斗统计
- 技能/物品面板切换逻辑适配

### Phase 4：后端数据补充 ✅
- `MonsterInstance.to_dict()` 新增 `level`、`monster_type`、`next_action` 字段
- `_build_state()` 在玩家回合时预计算怪物意图（调用 `decide_action`）
- 战斗开始 API 同样返回怪物意图数据
