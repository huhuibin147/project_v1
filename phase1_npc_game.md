# Phase 1：LLM NPC 像素风 RPG 对话系统

## 项目简介

一个基于 Web 的像素风 RPG 小游戏，玩家在村庄中探索，与由 LLM 驱动的多个 NPC 进行自由对话。NPC 具备独立人格、多轮对话记忆、意图识别、好感度系统、物品交易系统。玩家可选择职业，拥有角色属性和等级成长。

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | HTML5 Canvas + 原生 JS | 像素风渲染，无框架依赖 |
| 后端 | Python FastAPI | 异步接口，自带 API 文档 |
| LLM | OpenAI 兼容协议 | 支持任意兼容 OpenAI 接口的服务 |

## 快速开始

### 1. 配置 LLM

首次运行会自动从模板生成 `config.json`：

```bash
cd backend
python main.py
```

程序会提示：

```
==================================================
已创建 config.json，请编辑填入你的 API 配置：
  文件路径：/xxx/project_v1/config.json

  需要填写：
    "api_key"   — 你的 API Key
    "base_url"  — API 地址
    "model"     — 模型名称

填写完成后，重新执行启动命令即可。
==================================================
```

按提示编辑 `config.json`，填入实际配置：

```json
{
  "api_key": "你的 API Key",
  "base_url": "https://api.openai.com/v1",
  "model": "gpt-4o-mini"
}
```

> `config.json` 已加入 `.gitignore`，不会被提交到 Git，防止 API Key 泄漏。

支持任何 OpenAI 兼容接口，例如：

| 服务 | base_url 示例 |
|------|--------------|
| OpenAI | `https://api.openai.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| 本地 Ollama | `http://localhost:11434/v1` |
| 本地 vLLM | `http://localhost:8080/v1` |

### 2. 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

### 3. 启动服务

```bash
python main.py
```

首次运行会自动生成 `config.json` 并提示填写，填好后再次执行 `python main.py` 即可启动。

服务启动后访问 `http://localhost:8000`。

### 4. 游戏操作

| 按键 | 功能 |
|------|------|
| W / ↑ | 向上移动 |
| S / ↓ | 向下移动 |
| A / ← | 向左移动 |
| D / → | 向右移动 |
| E | 与 NPC 交互（需靠近，弹出对话/商店选项） |
| I | 打开/关闭背包面板 |
| P | 打开/关闭角色信息面板 |
| ESC | 关闭当前面板 |

## 使用方法

### 启动后你会看到什么

启动服务后浏览器打开 `http://localhost:8000`，你会看到一个像素风的村庄俯视角地图：

```
┌─────────────────────────────────────┐
│  🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲  │
│  🌲   🏠铁匠铺    ─ 道路 ─  🏠   🌲│
│  🌲   [老王]      │      │  🏠   🌲│
│  🌲              ─ ┘      └ ─     🌲│
│  🌲              │          │     🌲│
│  🌲    🚧栅栏    ─ ─ ─ ─ ─ ─     🌲│
│  🌲              🪨                🌲│
│  🌲    🧑 你在这里                  🌲│
│  🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲🌲  │
└─────────────────────────────────────┘
```

- 左上角显示操作提示
- 铁匠铺内站着 NPC「老王」，头顶有名字标签
- 你（蓝色角色）出生在村庄道路上

### 第一步：移动角色

用 **WASD** 或 **方向键** 控制角色上下左右移动。角色会朝移动方向改变面朝方向，移动时有走路动画。

注意：树木、房屋、栅栏、石头等障碍物无法穿越，会被自动挡住。

### 第二步：找到铁匠老王

老王站在铁匠铺内（地图左侧的棕色小屋）。走近他，当距离足够近时，他头顶会出现黄色提示条：**「按 E 交互」**。

### 第三步：与 NPC 交互

按下 **E** 键，屏幕中央弹出交互选项面板，提供两个按钮：**「对话」** 和 **「商店」**。点击「对话」打开对话面板：

```
┌──────────────────────────────────────────┐
│ 铁匠老王  情绪: 平静 | 好感度: 50/100 [商店] [×] │
├──────────────────────────────────────────┤
│ 老王：哟！冒险者，来俺这铁匠铺看看？       │
│      最近俺刚打了一批新货，包你满意！       │
│                                          │
│                                          │
├──────────────────────────────────────────┤
│ [输入你想说的话...                    ] [发送] │
└──────────────────────────────────────────┘
```

对话面板右上角有**「商店」**按钮，点击即可打开该 NPC 的商店面板。

### 第四步：与 NPC 自由对话

在输入框中输入你想说的话，按 **Enter** 或点击 **发送**。老王会根据你的话做出回复，回复时会显示「老王正在思考...」的加载提示。

**对话示例**：

| 你说的话 | 老王的回复（示例） | 意图识别 |
|---------|-------------------|---------|
| 你好啊老王 | 哈哈！你好你好，来来来，看看俺的新货！ | 闲聊 |
| 最近有什么需要帮忙的吗？ | 唉，别提了，村外最近闹怪物，好几个村民受伤了... | 任务 |
| 你这里有什么武器卖？ | 嘿嘿，俺这好东西多着呢！铁剑、精钢剑、铁匕首... | 交易 |
| 我想买一把铁剑 | 行！铁剑 80 金币，收你 80 金币。 | 交易（自动购买） |
| 我卖你一张狼皮 | 狼皮不错，20 金币收了！ | 交易（自动出售） |
| 你在这里多久了？ | 20 年啦！俺年轻时走南闯北，最后在这落了脚... | 闲聊 |
| 我偷了你的锤子 | 什么？！你小子敢偷俺的东西？！给俺还回来！ | 未知 |

### 第五步：观察 NPC 状态变化

对话面板顶部实时显示老王的状态：

- **情绪**：会随对话变化（平静 → 高兴 → 担忧 → 生气 等）
- **好感度**：初始 50 分，范围 0-100
  - 好感度上升：夸赞他的手艺、答应帮忙、友好问候
  - 好感度下降：侮辱他、偷东西、态度恶劣
  - 好感度到 100：老王对你非常信任，可能透露隐藏信息
  - 好感度到 0：老王拒绝和你说话

每次回复后，对话面板右侧会短暂显示**意图标签**（闲聊/任务/交易），2 秒后自动消失。

### 第六步：购买物品

在对话中表达购买意图，LLM 会自动识别并执行交易：

- 说「我想买一把铁剑」→ 自动扣除金币，铁剑放入背包
- 说「有没有药水卖」→ 老王会介绍商品，可以继续说「买两个」
- 交易成功后，回复中会显示 `【好嘞！1 个铁剑，收你 80 金币。】`

### 第七步：出售物品

在对话中表达出售意图：

- 说「我卖你一张狼皮」→ 自动从背包扣除，获得金币
- 说「我这有铁矿石，收不收」→ 老王会给出收购价

### 第八步：打开商店面板

靠近 NPC 后按 **E**，在交互选项面板中选择**「商店」**，即可打开该 NPC 的商店面板。

商店面板功能：

- 显示 NPC 商店库存（可购买的物品）
- 右上角显示双方金币
- 每个物品有「购买」按钮
- 你持有的物品会额外显示「出售」按钮
- 交易结果实时反馈

### 第九步：查看背包

按 **I** 键打开背包面板：

- 显示所有持有物品、数量和描述
- 右上角显示当前金币和「商店」入口按钮
- 物品按类型分色：武器(红)、防具(蓝)、消耗品(绿)、工具(黄)、材料(紫)

### 第十步：关闭面板

按 **ESC** 键关闭当前面板（对话/背包/商店），回到游戏画面。

再次靠近老王按 E 可以继续之前的对话，老王会**记住**之前和你聊过的内容（最近 10 轮对话）。

### 第十一步：找到杂货婆刘婶

刘婶站在杂货铺内（地图右侧的房屋）。走近她按 **E** 选择「对话」或「商店」，她卖的是日用品：食物、布料、绷带、蜡烛等。

每个 NPC 有独立的商店库存、收购仓库和对话历史，但玩家使用**统一的全局背包和金币**，无论在老王还是刘婶处交易，钱包和物品都是同一个。

**注意**：NPC 从玩家处收购的物品会存入独立的收购仓库（`acquired_inventory`），不会混入商店库存，确保各 NPC 商店物品的独立性。

### 第十二步：查看角色信息

按 **P** 键打开角色信息面板：

- 显示职业、等级、头像
- HP 条和 EXP 条
- 攻击、防御、速度属性
- 当前状态效果

游戏顶部 HUD 栏也实时显示等级和 HP。

### 对话技巧

1. **探索任务线**：反复问「有什么需要帮忙的」「村外怪物是什么情况」，老王会逐步透露更多信息
2. **提升好感度**：夸他的铁匠铺、表达愿意帮忙、对他友好
3. **触发交易**：问「有什么武器」「想买把剑」等关键词
4. **测试角色一致性**：骂他、说奇怪的话，看他如何反应（他会生气）
5. **多轮深聊**：连续追问同一话题，老王会基于之前的对话给出更深入的回答
6. **经济策略**：低价收购材料（狼皮、铁矿石），高价出售给老王赚差价
7. **多 NPC 交互**：不同 NPC 有不同性格和商品，多找几个人聊聊

### 查看 API 文档

启动后访问 `http://localhost:8000/docs` 可查看 FastAPI 自动生成的 Swagger API 文档，可以直接在页面上测试接口。

### 停止服务

在终端按 `Ctrl + C` 即可停止后端服务。

## 架构设计

### 整体流程

```
玩家移动 → 靠近 NPC → 按 E 对话 → 输入文字
    ↓
前端 POST /api/chat（携带 npc_id）
    ↓
后端 NPC Agent 构建 Prompt（角色设定 + 记忆 + 商店库存 + 输入）
    ↓
调用 LLM（OpenAI 兼容接口）
    ↓
解析 LLM 返回的 JSON（reply / intent / mood / affinity_change / trade_action）
    ↓
如果有 trade_action → 自动执行交易 → 更新背包和金币
    ↓
更新 NPC 状态 + 返回前端（含交易结果）
    ↓
前端显示回复 + 更新情绪/好感度/金币显示
```

### NPC Agent 设计

**多 NPC 系统**：每个 NPC 有独立的角色设定、性格、商店库存和存档。

| NPC | 身份 | 位置 | 商品 |
|-----|------|------|------|
| 铁匠老王 | 武器商人 | 铁匠铺 | 武器、防具、工具、药水 |
| 刘婶 | 日用品商人 | 杂货铺 | 食物、布料、绷带、蜡烛 |

**核心模块**：

| 模块 | 说明 |
|------|------|
| 对话记忆 | 滑动窗口保留最近 10 轮对话，注入 LLM 上下文 |
| 意图识别 | LLM 自动判断：chat（闲聊）、quest（任务）、trade（交易） |
| 好感度 | 0-100 分，根据玩家言行动态变化 |
| 情绪系统 | NPC 情绪随对话内容变化（平静/高兴/担忧/生气等） |
| 物品系统 | 玩家背包 + NPC 商店库存，支持对话驱动和 UI 直接交易 |
| 玩家档案 | 职业选择、等级经验、属性成长、HP/EXP 条 |

**Prompt 结构**：

```
System: 角色设定 + 当前状态（情绪/好感度）+ 商店库存信息 + 对话历史 + 可用物品ID列表
User: 玩家输入
→ LLM 返回结构化 JSON（含 trade_action）
```

### 玩家档案系统

**职业选择**：3 种职业，各有不同的基础属性和成长方向。

| 职业 | HP | 攻击 | 防御 | 速度 | 定位 |
|------|-----|------|------|------|------|
| 战士 | 120 | 15 | 12 | 8 | 高血量高防御 |
| 盗贼 | 80 | 18 | 6 | 15 | 高攻高速 |
| 法师 | 70 | 22 | 5 | 10 | 高攻低防 |

**等级系统**：
- 每次升级：HP+10、攻击+3、防御+2、速度+1
- 升级所需经验递增（exp_to_next × 1.5）
- 数据持久化到 `data/player_save.json`

### 前端渲染

**地图**：25×18 瓦片网格，每格 32×32 像素，包含 10 种瓦片类型（草地、泥土路、房屋墙、屋顶、树木、水面、铁匠铺地板、栅栏、石头、杂货铺地板）。

**碰撞检测**：基于瓦片类型的碰撞数组，玩家不可穿越障碍物。

**像素风绘制**：所有角色和地图元素通过 Canvas API 逐像素绘制，无外部图片资源依赖。NPC 和玩家按 Y 坐标排序绘制，实现遮挡效果。

## API 接口

### POST /api/chat

NPC 对话接口（支持交易意图自动执行）。

**请求**：
```json
{
  "message": "我想买一把铁剑"
}
```

**响应**：
```json
{
  "reply": "行！铁剑 80 金币，给你！\n【好嘞！1 个铁剑，收你 80 金币。】",
  "intent": "trade",
  "mood": "高兴",
  "affinity": 55,
  "trade": true,
  "player_inventory": [...],
  "player_gold": 120,
  "shop_inventory": [...],
  "shop_gold": 580
}
```

### GET /api/inventory

获取玩家全局背包与金币。

### GET /api/shop?npc_id=xxx

获取指定 NPC 商店库存。

### POST /api/trade

直接交易接口（不经过 LLM）。需要传 `npc_id`。

**请求**：
```json
{"action": "buy", "item_id": "iron_sword", "quantity": 1, "npc_id": "blacksmith"}
```

### GET /api/npc/status?npc_id=xxx

获取 NPC 当前状态（含金币信息）。

### GET /api/npc/config?npc_id=xxx

获取 NPC 配置信息。

### GET /api/npcs

获取所有可用 NPC 列表。

### GET /api/player

获取玩家档案信息（职业、等级、属性、HP/EXP）。

### GET /api/player/classes

获取所有可选职业列表。

### POST /api/player/update

更新玩家信息（设置名字、选择职业）。

**请求**：
```json
{"name": "冒险者", "class_id": "warrior"}
```

### POST /api/player/heal

玩家回复 HP。

### GET /

返回游戏页面（前端）。

### GET /docs

FastAPI 自动生成的 API 文档（Swagger UI）。

## 扩展方向

在 Phase 1 基础上，后续可扩展：

- **记忆系统**：向量数据库存储长期记忆，NPC 能记住几天前的对话
- **更多 NPC**：酒馆老板、村长、猎人等
- **任务系统**：NPC 下发任务，玩家完成后好感度提升，解锁新对话
- **装备系统**：物品可装备到角色身上，影响属性
- **传闻系统**：NPC 间信息传递，玩家行为影响全村态度
- **日程系统**：NPC 按时间切换行为（白天营业、晚上回家）
- **情绪动画**：NPC 头顶显示情绪图标（高兴/生气/害怕）
